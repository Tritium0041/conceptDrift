# 产品需求文档 (PRD)：ConceptDrift — 开发者灵感生成器

## 1. 产品概述
- **产品名称**：ConceptDrift
- **核心目标**：通过广泛采集多元灵感来源，自动生成包含技术可行性与市场新颖性评估的综合报告，帮助个人开发者快速发现有潜力的项目方向。
- **目标用户**：个人开发者、独立开发者、创客。

## 2. 核心场景与功能
### 2.1 典型使用场景
- **日常灵感获取**：开发者每天登录系统查看系统自动推送的最新灵感报告。
- **按需主动探索**：开发者在遇到创作瓶颈时，手动触发系统，要求针对特定方向或随机生成一个新的项目灵感及调研报告。
- **灵感归档与分享**：开发者将有价值的灵感报告导出为 Markdown 或 PDF 文件，用于本地存档或与他人讨论。

### 2.2 核心功能列表
1. **多元灵感采集引擎**
   - 支持多种维度的灵感来源，包括技术趋势（如 AI、Web3）、问题导向（如 HackerNews、Reddit 痛点挖掘）、市场空白分析以及跨领域概念的随机组合。
2. **自动化深度调研与报告生成**
   - **技术可行性评估**：评估实现难度、所需技术栈建议及开发周期估算。
   - **市场新颖性分析**：检索现有竞品、分析市场空白点及产品的差异化空间。
   - **商业潜力参考**：作为辅助项，提供目标用户群体和潜在变现路径的简要分析。
3. **灵感管理与导出**
   - 应用内存档：支持历史灵感报告的查看、搜索和管理。
   - 文件导出：支持将报告导出为 Markdown / PDF 格式。
4. **开放 API 接口**
   - 提供 RESTful API 或 GraphQL 接口，允许开发者通过代码获取灵感数据，便于未来集成到命令行工具（CLI）、IDE 插件或自动化工作流中。

### 2.3 非核心/排除功能
- 复杂的团队协作与评论功能（聚焦个人开发者）。
- 邮件或短信的自动推送（当前版本以应用内查看和导出为主）。

## 3. 约束与偏好
- **平台/载体**：优先开发 Web App（响应式设计），同时必须提供 API 接口。
- **设计风格/体验**：极客友好、简洁高效，报告排版需清晰易读（支持 Markdown 渲染）。
- **技术选型建议**：后端需具备较强的异步任务处理能力（因为调研和报告生成通常依赖 LLM 和搜索引擎，耗时较长）。

## 4. 后续建议
1. **数据源接入**：建议优先确定 2-3 个高质量的灵感数据源（如 GitHub Trending、Product Hunt、Reddit API）进行打样。
2. **LLM 提示词工程**：调研报告的质量强依赖于大模型的提示词设计，建议在开发初期重点测试“技术可行性”与“市场新颖性”的评估 Prompt。
3. **API 设计**：在开发 Web App 时，采用前后端分离架构，直接利用设计好的 API 接口为前端供数据，做到 API First。
